1、进程、线程、python中的协程

  • 进程是程序的执行实例，一个进程可以拥有多个线程，
  • 线程是进程中的一条执行流程
  1. 进程（Process）：
    • 进程是操作系统中资源分配的最小单位。每个进程都有独立的内存空间和系统资源，包括打开的文件、网络连接等。
    • 进程之间的切换开销较大，因为切换时需要保存和恢复多个资源。
    • 多进程适合在多核处理器上充分利用硬件资源，可以并行执行多个任务。
  2. 线程（Thread）：
    • 线程是进程内的执行单元，多个线程共享进程的内存空间和系统资源。
    • 线程之间切换的开销较小，因为它们共享进程的资源，切换时只需切换执行上下文。
    • 多线程适合在单个进程内执行多个任务，能够提高程序的效率和响应性。
  3. 协程（Coroutine）：
    • 协程是一种轻量级的并发编程方式，它在单个线程内支持多个执行流程，由程序员显式地控制切换。
    • 协程可以通过“挂起”和“恢复”来实现切换，切换开销很小，适合处理IO密集型任务。
    • Python的协程通过asyncio库实现，可以在异步编程中实现非阻塞的任务处理。

进程的创建过程需要分配什么资源？

  1. 内存空间： 每个进程都需要有自己的内存空间来存储代码、数据和堆栈。操作系统会分配一块内存空间供进程使用。
  2. 程序计数器： 进程需要一个独立的程序计数器来记录它当前执行的指令位置。
  3. 文件描述符： 进程可能需要访问文件、设备和网络资源。操作系统会为进程分配文件描述符，用于管理这些资源的访问。
  4. 进程标识符（PID）： 每个进程都有一个唯一的进程标识符，以便操作系统能够识别和管理不同的进程。
  5. 寄存器和CPU状态： 进程的寄存器和CPU状态需要在进程切换时进行保存和恢复。
  6. 优先级和调度信息： 操作系统需要为进程分配优先级和调度信息，以决定进程在CPU上的运行顺序。
  7. 文件描述符表： 进程需要一个独立的文件描述符表来管理它的打开文件和资源。
  8. 堆栈： 进程需要有自己的堆栈空间，用于存储函数调用、局部变量等信息。
  9. 进程控制块（PCB）： 操作系统会为每个进程创建一个进程控制块，用于存储进程的状态信息、上下文和资源分配情况。

什么是死锁？如何避免？

死锁只有同时满足以下四个条件才会发生：

  • 互斥条件；
  • 持有并等待条件；
  • 不可剥夺条件；
  • 环路等待条件；

最常见的解决思路是破坏循环等待条件，最简单的方法就是资源有序分配。

计算机网络

  • 说说OSI七层网络模型都有哪些层？他们都是什么作用？列举每一层都有哪些协议？
  1. 物理层（Physical Layer）：
    • 作用：传输原始比特流，负责物理介质的传输和接收。
    • 协议：Ethernet、USB、Bluetooth、RS-232等。
  2. 数据链路层（Data Link Layer）：
    • 作用：将比特流组织成帧，提供物理地址寻址和错误检测。
    • 协议：Ethernet、Wi-Fi、PPP（Point-to-Point Protocol）、HDLC（High-Level Data Link Control）等。
  3. 网络层（Network Layer）：
    • 作用：处理数据包在网络中的路径选择和逻辑寻址，实现跨网络通信。
    • 协议：IP（Internet Protocol）、ICMP（Internet Control Message Protocol）、OSPF（Open Shortest Path First）、BGP（Border Gateway Protocol）等。
  4. 传输层（Transport Layer）：
    • 作用：提供端到端的通信，处理数据的分段和重组，确保数据可靠传输。
    • 协议：TCP（Transmission Control Protocol）、UDP（User Datagram Protocol）、SCTP（Stream Control Transmission Protocol）等。
  5. 会话层（Session Layer）：
    • 作用：管理通信会话，建立、维护和终止数据交换。
    • 协议：NetBIOS、RPC（Remote Procedure Call）等。
  6. 表示层（Presentation Layer）：
    • 作用：处理数据格式的转换、加密、压缩，确保不同系统间的数据格式兼容。
    • 协议：TLS/SSL、JPEG、MPEG、ASCII等。
  7. 应用层（Application Layer）：
    • 作用：提供用户应用程序与网络的接口，支持各种应用，如电子邮件、网页浏览等。
    • 协议：HTTP、FTP（File Transfer Protocol）、SMTP（Simple Mail Transfer Protocol）、DNS（Domain Name System）等。
  • 讲讲HTTP和HTTPS的区别和联系？

HTTPS是在HTTP基础上增加了SSL/TLS安全验证，报文不再是明文传输，客户端与服务端通过协商秘钥，将报文经过秘钥加密后传输。

  • HTTPS加密方式是什么？

通过SSL/TLS进行加密，加密过程有四次握手，通过非对称加密的方式协商出来一个秘钥，后续的TCp报文传输都利用该秘钥进行对称加密。

  • 说说TCP协议的三次握手和四次挥手过程，为什么握手不能是两次，挥手不能是三次？

  • TCP保证可靠传输的方法？

  1. 确认和序列号： 每个发送的TCP段都有一个序列号，接收端使用这些序列号来确认接收到的段并按正确的顺序重新组装它们。如果发送端未收到确认，它会重新发送相应的数据。
  2. 确认机制： 接收方会发送确认段，告知发送方已成功收到数据。如果发送方未收到确认，它会重新发送数据。
  3. 超时重传： 发送方在发送数据后启动定时器。如果在定时器过期前未收到确认，发送方会重新发送数据。
  4. 滑动窗口： TCP使用滑动窗口来控制发送方发送多少数据，以避免接收方的缓冲区溢出。
  5. 流量控制： TCP使用流量控制机制来确保发送方不会发送过多的数据，从而导致接收方缓冲区溢出。接收方使用窗口大小来通知发送方可接受的数据量。
  6. 拥塞控制： TCP使用拥塞控制机制来避免网络拥塞。它通过监测网络的拥塞情况来调整发送速率。
  7. 连接管理： TCP使用三次握手建立连接和四次挥手关闭连接，确保双方通信的稳定和有序。

通过这些机制，TCP能够在不可靠的IP网络上实现可靠的数据传输，确保数据的完整性和顺序性。

  • TCP的拥塞控制如何实现？
  1. 慢启动（Slow Start）： 在连接刚开始时，发送方会以一个较小的拥塞窗口开始发送数据。随着每次成功接收到确认，窗口大小会指数增加，从而逐渐增加发送速率，直到达到一个阈值。
  2. 拥塞避免（Congestion Avoidance）： 一旦窗口大小达到阈值，发送方会进入拥塞避免阶段。在这个阶段，窗口大小以线性增长，而不是指数增长，以更稳定的速率发送数据。
  3. 快速重传和快速恢复： 如果发送方连续收到三个重复的确认，它会认为网络出现拥塞。此时，它会立即重传丢失的段，而不是等待超时。快速恢复阶段会让窗口大小减半，然后从此处重新开始增加。
  4. 超时重传： 如果发送方在一个特定的时间内未收到确认，它会认为出现了网络拥塞，触发超时重传，即重新发送未确认的数据段。

这些机制结合起来，使得TCP能够在网络出现拥塞时降低发送速率，从而避免过载网络。当网络拥塞减轻后，TCP会逐渐增加发送速率，以确保网络资源的有效利用和数据的可靠传输。
